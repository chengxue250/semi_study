This work maps Newton-iteration integer division to CUDA for precisions from 2^15 to 2^18 bits, beyond the division range supported by CGBN. You Only Index Once: Cross-layer sparse attention with shared routing You Only Index Once: Cross-layer sparse attention with shared routing

Y. Sun, Y. Zhang, L. Dong, et al.

arXiv:2606.06467 · 2026-06-04

Cross-Layer Sparse Attention computes token-level top-k routing once and reuses the index across decoder layers in KV-sharing architectures such as YOCO. That amortizes a cost that often erases the practical benefit of token-sparse attention while also reducing prefill, cache, and decode pressure. The result depends on architectures that already share KV state, so portability to conventional transformers remains the key caveat. Vortex: Programmable sparse-attention serving for AI agents Vortex: Programmable sparse-attention serving for AI agents

Z. Chen, X. Zhong, Q. Feng, et al.

arXiv:2606.06453 · 2026-06-04

Vortex combines a Python-embedded sparse-attention language with a page-centric tensor abstraction and a serving-stack backend. Algorithms generated and refined through the system reach up to 3.46 times the throughput of full attention while preserving accuracy in the reported tests. The preprint's main contribution is deployability and iteration speed, not a new attention rule. Agent Memory: System implications of stateful long-horizon workloads Agent Memory: System implications of stateful long-horizon workloads

Y. Omri, Z. Gan, Z. Broveak, et al.

arXiv:2606.06448 · 2026-06-04

This paper profiles ten agent-memory systems and attributes cost separately to memory construction, retrieval, and generation. TokenMizer: Graph-structured memory for long LLM sessions TokenMizer: Graph-structured memory for long LLM sessions

S. Mishra

arXiv:2606.06337 · 2026-06-04

TokenMizer stores session history as a typed knowledge graph and serializes it into compact resume blocks rather than carrying flat transcripts forward. On 21 controlled sessions, the blocks average 78 tokens versus 159 to 170 for the evaluated baselines while improving decision recall by 9 to 17 percentage points. The benchmark is small and controlled, so production-scale extraction cost and error accumulation remain open questions. Tangram makes non-uniform KV caches practical for multi-turn serving Tangram makes non-uniform KV caches practical for multi-turn serving

H. Kim, M. Kim, H. Kim, et al.

arXiv:2606.06302 · 2026-06-04

Tangram assigns deterministic per-head cache budgets, groups heads with similar retention needs into vectorized page tables, and balances work ahead of time. The design targets fragmentation, scheduling overhead, and poor kernel utilization that can cancel the memory savings of non-uniform KV compression. It is a serving-system proposal, and the supplied abstract does not expose the full end-to-end benchmark results.
